Finny — Deploy & MCP Bridge Handoff

Date: 2026-05-24 · Repo: Postergully/finny-claude-plugin · Public endpoint: https://finny.prod.11mirror.com/mcp · Status: live & verified

What was built

Forked the working lolly-claude-plugin (an OpenClaw/NetSuite MCP bridge + cowork plugin) into finny-claude-plugin, repointed at the Hermes Bedrock agent on EC2, and exposed the bridge over OAuth-protected HTTPS so Claude.ai Custom Connectors and Claude Code can both invoke finny_* tools.

No bridge logic was rewritten — the rename is mechanical (lolly→finny, openclaw→hermes) and only the upstream URL changed (:18789 openclaw gateway → :8642 Hermes API server). All five intent-validated tools and the cowork plugin's skills carry over verbatim.

Deploy flow (recap)

  1. Phase 1 (Mac) — fork lolly → finny via mechanical sed pass; git mv directories; switch plugin/.mcp.json to remote HTTP; add deploy/ artifacts; build & test (421/421 pass); push.
  2. Phase 3 (AWS) — extend HermesDeploy IAM with EIP + Route 53 statements; allocate & associate Elastic IP; open SG :80/:443; create A record.
  3. Phase 4 (EC2) — install Node 22, pnpm 9, Caddy 2.11; deploy key on GitHub for /opt/finny clone; pnpm install && pnpm build.
  4. Phase 5 (EC2) — generate API_SERVER_KEY + MCP_CLIENT_SECRET on the box; write .env files locked to ubuntu user.
  5. Phase 6 (EC2) — install three systemd units; ACME http-01 cert obtained in 3s; smoke /health, /v1/models, OAuth metadata.
  6. Phase 7 (EC2) — reboot test; all three services autostart in <5s.
  7. Phase 8 (Claude.ai) — Custom Connector form (URL + Client ID finny + Client Secret); auth handshake; first finny_query call returned a Bedrock-backed response.

Lessons learned (the parts the plan didn't predict)

  • Doc rename collateral damage. The bulk sed pass rewrote every lolly/openclaw string — including the design and plan docs that describe the rename. Restored those two files from pre-rename HEAD; future renames should explicitly exclude docs/superpowers/.
  • The .yml file extension trap. The grep filter used --include="*.yaml" but the repo had Docker Compose, Taskfile, and GitHub Actions in .yml. CI workflows would have broken silently until the next push. Always run a second pass or expand the glob.
  • Hermes API server isn't OpenAI-compat by default. Setting API_SERVER_ENABLED=true isn't enough — the Hermes gateway only logs the [API Server] line if it actually picks up the env. Verify by curling /v1/health not by trusting startup logs.
  • MCP SDK 1.29 host validation runs in two places. StreamableHTTPServerTransport validates the Host header, AND createMcpExpressApp applies another global host middleware. Both need allowedHosts set or you get Invalid Host on every endpoint. Wired both via a single MCP_ALLOWED_HOSTS env.
  • WWW-Authenticate must include resource_metadata per RFC 9728. Without it, Claude.ai's OAuth client can't discover the auth server from a 401 and fails with a generic start_error in the connector wizard. Pass resourceMetadataUrl to requireBearerAuth.
  • SDK dropped .includes() for .some(redirectUriMatches). The bridge's ALLOW_ANY_REDIRECT Proxy hack only intercepts .includes(), so v1.29 falls through to "Unregistered redirect_uri". Fix: set MCP_REDIRECT_URIS explicitly to the Claude.ai callback (https://claude.ai/api/mcp/auth_callback).
  • Caddy file logging is sandboxed. The systemd unit's ProtectHome=/ProtectSystem=strict means even a chown caddy:caddy /var/log/caddy won't let it write there. Drop the file log {} block; journald captures everything Caddy emits.
  • SSM heredocs with sub-sudo are pain. sudo -u ubuntu tee <file in a JSON-encoded SSM command runs the redirect as root and the tee as ubuntu — permission denied. Either write as root then chown, or drop the heredoc.
  • EC2 reboot drops the public IP unless it's an Elastic IP. The CloudFormation template gave the instance a regular public IP. Allocating and associating an Elastic IP is what makes the DNS record stable across reboots. Free while attached.
  • 11mirror.com lives on GoDaddy; only prod.11mirror.com is in Route 53. Used finny.prod.11mirror.com instead of fighting cross-provider DNS. Single-record change, no GoDaddy edits, no NS delegation work.
  • IAM least-privilege gets chicken-and-egg-ish. Each phase needed perms not in the prior policy version; we ended up updating HermesDeploy twice (once for CFN/EC2/IAM/SSM, once for EIP+Route53). Document the next missing perm in the spec so the admin can grant once instead of three times.
  • The bridge's auth model is "pre-configured client + form-entered secret", not Dynamic Client Registration. Claude.ai Custom Connector form takes Client ID + Client Secret directly. DCR is the wrong default; we briefly enabled it then turned it back off.

What's NOT done yet (work for the next branch)

  1. Hermes agent identity / persona. Right now Finny still introduces itself as a "ShareChat NetSuite ERP assistant" because the cowork plugin's skill envelopes are the lolly originals (kept verbatim by user direction). Set up Hermes' SOUL.md and persona CLI on the EC2 to give Finny its own voice and identity. User-owned task: do this via hermes CLI on the EC2.
  2. External memory: hindsight integration. Wire Hermes' memory provider to hindsight (replace the built-in in-memory store). Configure in ~/.hermes/config.yaml; verify cross-session recall via hermes doctor.
  3. Skill content rewrite. The 5 plugin skills (intent-decomposer, judging-output, day_dream, finny-usage, cowork-init) and the 4 finance subskills still describe NetSuite-specific operations. Either rewrite for Finny's domain or document the mapping so users know what NetSuite-language inside the skills actually triggers in Finny's domain.
  4. Finance skill domain-fit decision. Same as above — the finance/ subskills (audit-netsuite, income-statement, reconciliation, variance-analysis) carry NetSuite assumptions. Decide whether Finny is also a NetSuite-aware agent or whether to retire/rewrite that skill.
  5. Hermes systemd unit hygiene. Hermes warns Stale systemd unit detected: TimeoutStopSec=90s but drain_timeout=180s. Could SIGKILL during graceful drain. Either bump TimeoutStopSec=210s in deploy/systemd/hermes-api.service or run hermes gateway service install --replace.
  6. MCP_REDIRECT_URIS is broad. Currently allowlist includes http://localhost and http://127.0.0.1 for Inspector/dev. Tighten to just https://claude.ai/api/mcp/auth_callback and https://claude.com/api/mcp/auth_callback in production.
  7. Observability gaps. No CloudWatch metrics, no uptime monitoring, no alerting. Service logs are in journald. Caddy's access log was disabled. Add at least: an external uptime check (UptimeRobot or CloudWatch Synthetics on /health) before relying on this in prod.
  8. Secret rotation cadence. Rotate MCP_CLIENT_SECRET and API_SERVER_KEY on a schedule (every 90 days?). Both can be rotated via SSM in <30s; document the runbook so it's not tribal knowledge.
  9. Branch + PR discipline. All Phase 1 work happened on main by user permission to bootstrap. New work should branch and PR.

Quick-reference runbook

Connect to the EC2

aws ssm start-session --target i-0ef58962b09d490ee --region us-east-1
sudo -iu ubuntu

Service control

sudo systemctl status hermes-api finny-mcp caddy
sudo systemctl restart finny-mcp
sudo journalctl -u finny-mcp -f

Deploy a code change to the bridge

# on Mac
cd /Applications/finny-claude-plugin
git checkout -b feat/whatever
# ...edit, commit...
git push origin feat/whatever
gh pr create --fill
# After merge to main, on EC2:
cd /opt/finny && sudo -u ubuntu git pull
sudo -u ubuntu pnpm install --frozen-lockfile
sudo -u ubuntu pnpm build
sudo systemctl restart finny-mcp

Rotate a secret

# MCP_CLIENT_SECRET (re-paste new value into Claude.ai connector after)
NEW=$(openssl rand -hex 32)
sudo sed -i "s|^MCP_CLIENT_SECRET=.*|MCP_CLIENT_SECRET=$NEW|" /opt/finny/bridge/.env
sudo systemctl restart finny-mcp
echo "New secret: $NEW"

Smoke test from anywhere

curl -sI https://finny.prod.11mirror.com/health        # 200 OK
curl -sS https://finny.prod.11mirror.com/.well-known/oauth-authorization-server | jq .issuer
# expect: https://finny.prod.11mirror.com/
